Design System Components

Comprehensive guide to all UI elements following our design philosophy

Design Philosophy

Calm, intentional, coherent, confident interface elements that prioritize user focus and productivity. Our design avoids distractions with a clean aesthetic that guides users toward their goals.

Core Principles

  • No gradients, glassmorphism, neon colors, or animation noise
  • Maximum 4 colors across entire UI
  • Intentional whitespace using consistent spacing system
  • Typography hierarchy that guides attention effectively
  • Thoughtful interactions with appropriate transitions

Color System

Background: #F7F6F3
Text: #111111
Accent: #8B0000
Success: #4CAF50
Warning: #FF9800

Usage Guidelines

Use the background color consistently across all surfaces. Reserve the accent color for primary actions and important highlights. Success and warning colors should only appear in status indicators and feedback messages.

Body text (16px) - Clean, readable, with 1.6 line height. This demonstrates the sans-serif Inter font family used throughout the interface. The generous line height improves readability and creates a sense of spaciousness.

Caption text (12-14px) for additional information, metadata, or subtle guidance.

Usage Guidelines

Serif headings (Playfair Display) create distinction and gravitas for section titles and important labels. Sans-serif body text (Inter) ensures optimal readability for extended content. Maintain the 16-18px size range for comfortable reading.

Spacing System

8px (XS)
16px (SM)
24px (MD)
40px (LG)
64px (XL)

Usage Guidelines

Only use values from this scale. Never use arbitrary values like 13px or 27px. The progression creates rhythm and predictable relationships between elements. XS for micro-spacing, SM for component padding, MD for section separation, LG for major divisions, XL for page-level spacing.

Layout Structure

Every page follows the same fundamental layout structure:

  1. Top Bar: App name left, progress center, status right
  2. Context Header: Large serif headline with descriptive subtext
  3. Primary Workspace (70%): Main content area with clean cards
  4. Secondary Panel (30%): Contextual information and guidance
  5. Proof Footer: Completion checklist

Responsive Behavior

Below 900px, the primary and secondary panels stack vertically with the secondary panel appearing below the primary content.
